Site-Wide Issues
Cross-cutting issues that affect the entire website's search visibility
⚠ Critical: Single-Page App Architecture

All service sections share one URL — cannot rank individually

The website uses JavaScript to hide/show sections (Landing, Special Events, Hair, Nails, Nails, About, Booking). To Google's crawler, this is one page at one URL with all content present simultaneously. This means "Special Events" cannot rank separately from "Nail Services," which cannot rank separately from "Hair," etc. A user searching "gel nails Union MS" and a user searching "event makeup Union Mississippi" both land at the same undifferentiated URL.

Long-term fix: Convert each service category to its own HTML page (special-events.html, hair.html, nails.html, about.html) with unique URLs, titles, H1s, and meta descriptions. Short-term fix (no rebuild required): Add URL hash navigation (#nails, #hair, #special-events) so Google can at least identify and link to specific sections, and use History API pushState to create distinct-feeling URLs.
Critical

Five H1 tags coexist in the DOM simultaneously

The HTML source contains H1 tags for: "Special Events," "Hair," "Nails," "Book an Appointment," and "About Belle's" — all present in the DOM at once, even though only one section is visible at a time. Google reads all H1s and may be confused about the page's primary topic. The landing section (the most important) has no H1 at all.

Change the hidden section headings from H1 to H2 using JavaScript when those sections are hidden, or use CSS display:none on sections and ensure the landing section has a proper H1 tag. The simplest fix: add a visually hidden H1 to the landing section and downgrade the section headings to H2.

Warning
robots.txt & Sitemap

robots.txt is missing

No robots.txt file found in the project root. Without it, search engines have no explicit crawl instructions. When the site goes live, Google will attempt to access /robots.txt and receive a 404 error, which is a minor negative signal.

Create a simple robots.txt in the project root:
User-agent: *
Allow: /
Sitemap: https://yourdomain.com/sitemap.xml
Critical

sitemap.xml is missing

No XML sitemap found. A sitemap tells Google which pages exist and when they were last updated. For a local business site, it accelerates indexing and ensures all pages are discovered.

Create sitemap.xml once the site is deployed. For a single-page site:
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url><loc>https://yourdomain.com/</loc><lastmod>2026-06-08</lastmod></url>
</urlset>Submit via Google Search Console after deployment.
